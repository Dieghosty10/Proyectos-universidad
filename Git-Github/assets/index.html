¿Qué es Git y GitHub?

Git: El sistema de control de versiones

Git es un sistema de control de versiones distribuido creado por Linus Torvalds en 2005.

  • Distribuido: Cada desarrollador tiene una copia completa del historial.
  • Rápido: Eficiente en proyectos grandes.
  • Seguro: Usa SHA-1 para la integridad de los datos.
  • Ramificación poderosa: Manejo eficiente de ramas.
  • Historial completo de cambios.

Repositorio: Directorio gestionado por Git.
Commit: Instantánea de cambios.
Rama: Línea de desarrollo.
Stage/Index: Área de preparación.
Merge: Combina cambios de ramas.

GitHub: La plataforma de colaboración

GitHub es una plataforma de alojamiento para proyectos Git, adquirida por Microsoft en 2018.

  • Colaboración: Facilita el trabajo en equipo.
  • GitHub Pages: Hospedaje web estático.
  • Integración continua: Soporte para CI/CD.
  • GitHub Actions: Automatización de flujos.
  • Gestión de proyectos y revisión de código.

Diferencias clave

Git GitHub
Sistema de control de versiones Plataforma de alojamiento
Se instala localmente Servicio en la nube
Gratuito y open-source Planes gratuitos y de pago

Flujo de trabajo típico

  1. git clone [url] - Copiar un repositorio remoto
  2. git checkout -b [nombre-rama] - Crear una nueva rama
  3. git add [archivo] - Preparar cambios
  4. git commit -m "mensaje descriptivo" - Guardar cambios localmente
  5. git push origin [nombre-rama] - Enviar cambios a GitHub
  6. Crear Pull Request en GitHub
  7. Revisar y fusionar cambios

Comandos Git útiles

git status      # Estado del repositorio
git log         # Historial de commits
git diff        # Cambios no preparados
git pull        # Obtener cambios del remoto
git branch      # Listar ramas
git reset       # Deshacer cambios
